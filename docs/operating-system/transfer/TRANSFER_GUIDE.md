# Highway 38 Operating System — Transfer Guide

## Package boundaries

### Reusable Core Engine

- approval-gated queue model
- selected-row execution pattern
- canonical queue/status schemas
- proof and error logging requirements
- duplicate-action controls
- operator, developer, recovery, and maintenance documentation
- exported Apps Script source

### Customer Configuration Layer

- owner identity and approval wording
- Google resource IDs and folder structure
- Gmail sender/account
- time zone
- library and deployment settings
- repository and website URLs

### Branding and Templates

- business name, tagline, colors, logo, website copy
- form wording
- email/quote/follow-up templates
- service catalog, pricing, product definitions, and sample assets

### Highway 38 Business Pack

- Highway 38 Solutions brand
- Problem Snapshot and other Highway 38 service definitions
- Highway 38 pricing and package structure
- Highway 38 public website and samples
- Highway 38 customer communications

## Transfer procedure

1. Freeze the source commit and export the complete live Apps Script project.
2. Copy the Core Engine into a customer-owned repository and Drive structure.
3. Create customer-owned Form, Sheets, Apps Script, Gmail, Drive, Web App, and website resources.
4. Replace customer configuration values.
5. Apply customer branding and templates without changing safety logic.
6. Run function, menu, file, header, status, link, privacy, and secret checks.
7. Run blocked-row and duplicate-lock tests.
8. Run one controlled owner-approved internal-recipient test.
9. Deliver operator training and recovery instructions.
10. Transfer ownership/access and record customer acceptance.

## Transfer classifications

- **Fully transferable now:** documented approval model, queue/status design, operator controls, proof/error principles, selected-row rules, and current exported module.
- **Transferable with configuration:** Form, Sheets, Drive structure, Gmail, GitHub Pages, Web App, library, and customer tracker.
- **Highway 38-specific:** brand, website content, pricing, services, samples, product names, and customer templates.
- **Rick-account dependent:** current Google ownership, Gmail sender, Script ID, Web App deployment, GitHub ownership, and connected social tools.
- **Installer automation needed:** complete project export/import, resource creation, configuration injection, permission setup, Web App deployment, and automated acceptance test.
